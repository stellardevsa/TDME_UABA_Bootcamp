# Module 3 - Building a Marketplace App

> **Bootcamp Week 3 Tutorial**
> By the end of this tutorial you will have a working decentralised marketplace running on the Stellar testnet, and understand how to adapt it for mainnet with ZARC as the payment currency.

---
## 1. What We're Building

A minimal but fully functional decentralised marketplace where:

- Users connect a Stellar wallet directly in the browser.
- Products are listed with a price in a stablecoin (USDC on testnet, ZARC on mainnet).
- Clicking **Buy Now** builds and signs a real Stellar transaction, then broadcasts it to the network.
- No backend server is involved — everything runs in the browser and talks directly to Stellar's Horizon API.

This is a real Web3 DApp. The only centralised part is the Horizon API endpoint, and even that can be swapped for a self-hosted node.

### components/Product.jsx

```jsx
import {
    TransactionBuilder,
    Operation,
    Networks,
    BASE_FEE,
    Transaction,
    Asset,
} from '@stellar/stellar-sdk';

function Product({ product, publicKey, kit, server, setStatus }) {
    const handleBuy = async () => {
        try {
            setStatus(`Preparing payment for ${product.name}...`);

            const sourceAccount = await server.loadAccount(publicKey);

            const usdcAsset = new Asset(
                "USDC",
                "GBBD47IF6LWK7P7MDEVSCWR7DPUWV3NY3DTQEVFL4NAT4AQH3ZLLFLA5"
            );

            const tx = new TransactionBuilder(sourceAccount, {
                fee: BASE_FEE,
                networkPassphrase: Networks.TESTNET,
            })
                .addOperation(
                    Operation.payment({
                        destination: product.seller,
                        asset: usdcAsset,
                        amount: product.price,
                    })
                )
                .setTimeout(30)
                .build();

            setStatus('Waiting for wallet signature...');

            const { signedTxXdr } = await kit.signTransaction(tx.toXDR(), {
                networkPassphrase: Networks.TESTNET,
                address: publicKey,
            });

            const signedTx = new Transaction(signedTxXdr, Networks.TESTNET);

            setStatus('Submitting transaction to Stellar testnet...');

            const result = await server.submitTransaction(signedTx);

            setStatus(
                `Payment successful for ${product.name}. Tx: ${result.hash.slice(0, 12)}...`
            );
        } catch (error) {
            console.error('Purchase failed:', error);

            const errorMessage =
                error?.response?.data?.extras?.result_codes
                    ? JSON.stringify(error.response.data.extras.result_codes)
                    : error?.message || 'Unknown error';

            setStatus(`Purchase failed: ${errorMessage}`);
        }
    };

    return (
        <div className="product-card">
            <h3>{product.name}</h3>
            <p>Price: {product.price} USDC</p>
            <p>
                Seller: {product.seller.slice(0, 6)}...{product.seller.slice(-4)}
            </p>
            <button onClick={handleBuy}>Buy Now</button>
        </div>
    );
}

export default Product;
```

## 6. Key Library Deep Dives

### Stellar SDK

**npm package:** `@stellar/stellar-sdk`
**Official docs:** https://stellar.github.io/js-stellar-sdk/

The Stellar SDK is the bridge between your JavaScript app and the Stellar blockchain. Think of it as the toolkit that knows how to speak Stellar's language.

#### What it does in our app

| Class / Function | What it does |
|---|---|
| `Horizon.Server` | Creates a connection to a Horizon node (the Stellar API) |
| `server.loadAccount(publicKey)` | Fetches the account's current state (sequence number, balances) from the network — **required before building any transaction** |
| `TransactionBuilder` | A fluent builder that assembles a transaction step by step |
| `.addOperation(...)` | Adds an instruction to the transaction (e.g. "send 10 USDC to address X") |
| `Operation.payment(...)` | Constructs a payment operation |
| `Asset` | Represents a Stellar asset — either native XLM or a custom token like USDC or ZARC |
| `Networks.TESTNET` / `Networks.PUBLIC` | Constants for the network passphrase — prevents a testnet transaction from being replayed on mainnet |
| `BASE_FEE` | The minimum transaction fee (currently 100 stroops = 0.00001 XLM) |
| `.build()` | Finalises the transaction and produces a `Transaction` object |
| `tx.toXDR()` | Serialises the transaction to XDR format (a compact binary format Stellar uses) — needed before passing to the wallet for signing |
| `new Transaction(signedTxXdr, Networks.TESTNET)` | Deserialises the signed XDR back into a Transaction object |
| `server.submitTransaction(signedTx)` | Broadcasts the signed transaction to the network |

#### Transaction lifecycle

Every Stellar payment follows this exact sequence:

```
1. Load account (get sequence number)
        ↓
2. Build transaction (TransactionBuilder)
        ↓
3. Serialise to XDR (tx.toXDR())
        ↓
4. Sign with wallet (Stellar Wallets Kit)
        ↓
5. Deserialise signed XDR
        ↓
6. Submit to network (server.submitTransaction)
        ↓
7. Get transaction hash (result.hash)
```

#### Assets explained

Stellar supports thousands of custom tokens. An asset is identified by two things: a **code** (like "USDC") and an **issuer** (the Stellar address of the organisation that created the token).

```javascript
// Native XLM - no issuer needed
Asset.native()

// USDC on testnet - issued by Circle's testnet address
const usdcAsset = new Asset(
  "USDC",
  "GBBD47IF6LWK7P7MDEVSCWR7DPUWV3NY3DTQEVFL4NAT4AQH3ZLLFLA5"
);

// ZARC on mainnet (South African Rand stablecoin)
const zarcAsset = new Asset(
  "ZARC",
  "GAJKJQSVUUPKDWV6SS3COVVREPR5YGJEM2MHKG6AWPNCTX4IUFBPT6MU" // verify on stellar.expert
);
```

> ⚠️ **Important:** The issuer address is what makes an asset unique. `USDC` from Circle is completely different from a token called `USDC` issued by someone else. Always verify the issuer address before using any asset in production.

### Stellar Wallets Kit

**Website:** https://stellarwalletskit.dev
**Created by:** Creit Tech (creators of xBull Wallet)

#### Why do we need it?

Without Stellar Wallets Kit, you would need to write separate integration code for every wallet your users might have — Freighter has its own API, xBull has another, Albedo has another, WalletConnect is entirely different again. That is a lot of code to write and maintain.

Stellar Wallets Kit lets you forget about reading each wallet's individual documentation and instead support all of them at once with a single library.

It is framework agnostic, meaning it works with React, Angular, Vue, or even vanilla JavaScript.

#### Supported wallets

The kit supports xBull Wallet (both PWA and extension versions), Albedo, Freighter, Rabet (extension version), WalletConnect, Lobstr, Hana, Hot Wallet, and Klever Wallet.

#### How it works

The kit provides a static class `StellarWalletsKit` with three main responsibilities:

**1. Initialisation** - load all wallet modules once at app startup:
```javascript
StellarWalletsKit.init({
  modules: defaultModules(), // loads all standard wallet modules
});
```

**2. Authentication** - opens a modal where the user picks their wallet:
```javascript
const { address } = await StellarWalletsKit.authModal();
// address is the user's Stellar public key
```

**3. Signing** - passes an XDR transaction to whichever wallet the user connected:
```javascript
const { signedTxXdr } = await StellarWalletsKit.signTransaction(tx.toXDR(), {
  networkPassphrase: Networks.TESTNET,
  address: publicKey,
});
```

The beauty of this abstraction is that whether your user has Freighter or xBull installed, the `signTransaction` call works identically. The kit handles the wallet-specific communication internally.

Real-world projects using the Stellar Wallets Kit include the official Stellar Lab (lab.stellar.org), xBull Swap, and Blend Capital.

### components/Product.jsx - The Payment Engine

This is the most important component. Let's walk through `handleBuy` step by step.

#### Step 1 - Load the buyer's account

```javascript
const sourceAccount = await server.loadAccount(publicKey);
```

Before building a transaction, Stellar requires the current **sequence number** of the buyer's account. Every Stellar account has a sequence number that increments with each transaction — this prevents the same transaction from being submitted twice (replay protection). `loadAccount` fetches this live from the network.

#### Step 2 - Define the asset

```javascript
const usdcAsset = new Asset(
  "USDC",
  "GBBD47IF6LWK7P7MDEVSCWR7DPUWV3NY3DTQEVFL4NAT4AQH3ZLLFLA5"
);
```

We define the asset we want to pay with. The first argument is the **asset code** and the second is the **issuer** - Circle's testnet address for USDC. Both together uniquely identify the token.

#### Step 3 - Build the transaction

```javascript
const tx = new TransactionBuilder(sourceAccount, {
  fee: BASE_FEE,
  networkPassphrase: Networks.TESTNET,
})
  .addOperation(
    Operation.payment({
      destination: product.seller,
      asset: usdcAsset,
      amount: product.price,
    })
  )
  .setTimeout(30)
  .build();
```

Breaking this down:
- `TransactionBuilder(sourceAccount, ...)` - starts building a transaction from the buyer's account.
- `fee: BASE_FEE` - sets the minimum fee (100 stroops = 0.00001 XLM).
- `networkPassphrase: Networks.TESTNET` - cryptographically binds the transaction to testnet. Change this to `Networks.PUBLIC` for mainnet.
- `.addOperation(Operation.payment(...))` - adds the "send money" instruction. A single Stellar transaction can have up to 100 operations.
- `.setTimeout(30)` - the transaction is rejected by the network if not submitted within 30 seconds of being built.
- `.build()` - finalises the transaction object.

#### Step 4 — Sign with the wallet

```javascript
const { signedTxXdr } = await kit.signTransaction(tx.toXDR(), {
  networkPassphrase: Networks.TESTNET,
  address: publicKey,
});
```

`tx.toXDR()` converts the transaction to a compact binary-encoded string (XDR format). This string is passed to the user's wallet, which displays the transaction details, waits for user approval, and signs it with the user's private key. The private key **never leaves the wallet** - this is the core security model of all Stellar DApps.

#### Step 5 — Submit to the network

```javascript
const signedTx = new Transaction(signedTxXdr, Networks.TESTNET);
const result = await server.submitTransaction(signedTx);
```

We convert the signed XDR back into a `Transaction` object and submit it to Stellar's network via Horizon. `result.hash` is the unique transaction ID - you can look it up on [Stellar Expert](https://stellar.expert/explorer/testnet).

#### Error handling

```javascript
const errorMessage =
  error?.response?.data?.extras?.result_codes
    ? JSON.stringify(error.response.data.extras.result_codes)
    : error?.message || 'Unknown error';
```

Horizon returns structured error codes inside `error.response.data.extras.result_codes`. We prefer these over the generic `error.message` because they tell us exactly what went wrong on-chain (e.g. `op_no_trust`, `op_underfunded`).

ZARC (South African Rand Coin) is a ZAR-pegged stablecoin on the Stellar mainnet. Here is exactly what to change in each file to go live with ZARC.

### Change 1 — `App.jsx`: Switch the Horizon server

### Change 2 - `components/Product.jsx`: Switch the network passphrase and asset

Replace every instance of `Networks.TESTNET` with `Networks.PUBLIC`, and swap the asset definition:

```jsx
import {
    TransactionBuilder,
    Operation,
    Networks,
    BASE_FEE,
    Transaction,
    Asset,
} from '@stellar/stellar-sdk';

function Product({ product, publicKey, kit, server, setStatus }) {
    const handleBuy = async () => {
        try {
            setStatus(`Preparing payment for ${product.name}...`);

            const sourceAccount = await server.loadAccount(publicKey);

            // ✅ CHANGED: ZARC asset on mainnet
            const zarcAsset = new Asset(
                "ZARC",
                "GAJKJQSVUUPKDWV6SS3COVVREPR5YGJEM2MHKG6AWPNCTX4IUFBPT6MU"
            );

            const tx = new TransactionBuilder(sourceAccount, {
                fee: BASE_FEE,
                networkPassphrase: Networks.PUBLIC, // ✅ CHANGED
            })
                .addOperation(
                    Operation.payment({
                        destination: product.seller,
                        asset: zarcAsset,            // ✅ CHANGED
                        amount: product.price,
                    })
                )
                .setTimeout(30)
                .build();

            setStatus('Waiting for wallet signature...');

            const { signedTxXdr } = await kit.signTransaction(tx.toXDR(), {
                networkPassphrase: Networks.PUBLIC,  // ✅ CHANGED
                address: publicKey,
            });

            const signedTx = new Transaction(signedTxXdr, Networks.PUBLIC); // ✅ CHANGED

            setStatus('Submitting transaction to Stellar mainnet...');

            const result = await server.submitTransaction(signedTx);

            setStatus(
                `Payment successful for ${product.name}. Tx: ${result.hash.slice(0, 12)}...`
            );
        } catch (error) {
            console.error('Purchase failed:', error);

            const errorMessage =
                error?.response?.data?.extras?.result_codes
                    ? JSON.stringify(error.response.data.extras.result_codes)
                    : error?.message || 'Unknown error';

            setStatus(`Purchase failed: ${errorMessage}`);
        }
    };

    return (
        <div className="product-card">
            <h3>{product.name}</h3>
            <p>Price: {product.price} ZARC</p> {/* ✅ CHANGED label */}
            <p>
                Seller: {product.seller.slice(0, 6)}...{product.seller.slice(-4)}
            </p>
            <button onClick={handleBuy}>Buy Now</button>
        </div>
    );
}

export default Product;
```

## 11. Common Errors & Fixes

| Error | Cause | Fix |
|---|---|---|
| `ReferenceError: global is not defined` | Missing polyfill in Vite | Add `define: { global: 'window' }` to `vite.config.js` |
| `op_no_trust` in result codes | Buyer has no ZARC/USDC trustline | Ask buyer to add the asset to their wallet first |
| `op_underfunded` | Buyer doesn't have enough of the asset | Fund the testnet wallet; check balance on Stellar Expert |
| `tx_bad_seq` | Sequence number mismatch | You submitted the same transaction twice, or the account changed between build and submit. Rebuild the transaction fresh. |
| `Wallet connection was cancelled` | User closed the modal | Catch the error gracefully — this is expected user behaviour |
| `Transaction expired` | Took more than 30 seconds to sign | Increase `.setTimeout(60)` or ask the user to sign promptly |
| `op_no_destination` | The seller address does not exist on-chain | Verify the seller's Stellar address is active (has been funded) |
